# ⚖️ MedianFinder 详解

## ① 问题是什么？

不断往一个集合里加数字，随时能查到**中位数**。

```
中位数 = 把所有数从小到大排好, 正中间的那个
        (偶数个时 = 中间两个的平均)
```

例如: `[1, 3, 4, 1, 5]` 排序后 → `[1, 1, 3, 4, 5]`, 中位数 = **3**

---

## ② 核心思路 —— 一刀切成两半

> **关键观察**: 中位数只关心「中间」，不关心两端的顺序。

把排好序的数组从中间切开：

```
    ┌────────── 较小的一半 ──────────┐┌────────── 较大的一半 ──────────┐
    │                                ││                                │
    │  1    1    3                  ││  4    5                        │
    │                    ↑          ││  ↑                             │
    │           这半的最大值=3       ││  这半的最小值=4                 │
    └───────────────────────────────┘└────────────────────────────────┘
                   ↑                                    ↑
              只需要知道 3                只需要知道 4

    总数 = 5 (奇数) → 中位数 = 左边那半的最大值 = 3
```

**所以我们不需要完整的排序！只需要维护两个「堆」**：

| 堆 | 存什么 | 堆顶是什么 | 为什么用这种堆 |
|:--|:--|:--|:--|
| **大顶堆** (max_heap) | 较小的一半 | 这半的 **最大值** | 因为我们要随时取这半的最大值 |
| **小顶堆** (min_heap) | 较大的一半 | 这半的 **最小值** | 因为我们要随时取这半的最小值 |

---

## ③ 图示：堆顶天然就是中位数的候选

```
           ┌─────────────────────┐          ┌─────────────────────┐
           │   max_heap (大顶堆)  │          │   min_heap (小顶堆)  │
           │                     │          │                     │
           │       ┌─3─┐         │          │       ┌─4─┐         │
           │      ┌┘   └┐        │          │      ┌┘   └┐        │
           │     ┌1┐   ┌1┐       │          │     ┌4┐   ┌5┐       │
           │     └─┘   └─┘       │          │     └─┘   └─┘       │
           │                     │          │                     │
           │  堆顶 = 3 (最大值)   │          │  堆顶 = 4 (最小值)   │
           └─────────────────────┘          └─────────────────────┘
                      ↑                                   ↑
                      └───────────────┬───────────────────┘
                                      │
                              总数奇数 (5) → 中位数 = 3
                              总数偶数 (6) → 中位数 = (3+4)/2 = 3.5
```

**两个堆顶正好「夹住」中位数**。

---

## ④ 两个规则 —— 就这么简单

### 规则 1：两堆大小必须平衡

```
  len(max_heap) 要么等于 len(min_heap)  (偶数个)
                要么比它多 1              (奇数个)

  永远不允许 len(min_heap) > len(max_heap)
```

### 规则 2：所有较小半的数 ≤ 所有较大半的数

```
  max_heap 里的每个数  ≤  min_heap 堆顶
  min_heap 里的每个数  ≥  max_heap 堆顶
```

## ⑤ 一步一步走

### 初始状态

```
  已排序: [ ]
  
  ┌──────────┐     ┌──────────┐
  │ max_heap │     │ min_heap │
  │   (空)   │     │   (空)   │
  └──────────┘     └──────────┘
```

---

### Step 1: addNum(3)

> 第一个数，直接放进 max_heap

```
  已排序: [3]
          ↑
        中位数

  规则: max_heap 先收第一个数

  ┌──────────┐     ┌──────────┐
  │ max_heap │     │ min_heap │
  │   [3]    │     │   (空)   │
  │  堆顶=3  │     │          │
  └──────────┘     └──────────┘
  
  len(max)=1, len(min)=0  →  平衡 ✓

  中位数 = max_heap 堆顶 = 3
```

---

### Step 2: addNum(1)

> 1 ≤ max 堆顶(3) → 属于较小半 → 放 max_heap
> 放完后 max 比 min 多 2 个 → 不平衡！把 max 堆顶(3) 搬到 min

```
  已排序: [1, 3]
            ↑  ↑
           (1+3)/2 = 2

  ① 判断: 1 ≤ 3 → 放 max_heap      ② 平衡: max 多 2 个
  ┌──────────────┐  ┌─────────┐     ┌──────────┐  ┌──────────┐
  │   max_heap   │  │min_heap │     │ max_heap │  │ min_heap │
  │    [1, 3]    │  │  (空)   │ →   │   [1]    │  │   [3]    │
  │   堆顶 = 3   │  │         │     │  堆顶=1  │  │  堆顶=3  │
  └──────────────┘  └─────────┘     └──────────┘  └──────────┘
  len(max)=2, min=0 ✗               len(max)=1, min=1 ✓

  中位数 = (1 + 3) / 2 = 2.0
```

---

### Step 3: addNum(4)

> 4 > max 堆顶(1) → 属于较大半 → 放 min_heap
> 放完后 min 比 max 多 → 不平衡！把 min 堆顶(3) 搬到 max

```
  已排序: [1, 3, 4]
             ↑
           中位数=3

  ① 判断: 4 > 1 → 放 min_heap       ② 平衡: min 多 1 个
  ┌──────────┐  ┌──────────┐        ┌──────────┐  ┌──────────┐
  │ max_heap │  │ min_heap │        │ max_heap │  │ min_heap │
  │   [1]    │  │  [3, 4]  │   →    │  [1, 3]  │  │   [4]    │
  │  堆顶=1  │  │  堆顶=3  │        │  堆顶=3  │  │  堆顶=4  │
  └──────────┘  └──────────┘        └──────────┘  └──────────┘
  len(max)=1, min=2 ✗               len(max)=2, min=1 ✓

  中位数 = max 堆顶 = 3 (奇数个, max 多一个)
```

---

### Step 4: addNum(1)（又来了一个 1）

> 1 ≤ max 堆顶(3) → 放 max_heap
> max 多 2 个 → 把 3 搬到 min

```
  已排序: [1, 1, 3, 4]
            ↑     ↑
           (1+3)/2 = 2

  ① 判断: 1 ≤ 3 → 放 max           ② 平衡: max 多 2 个
  ┌──────────┐  ┌──────────┐        ┌──────────┐  ┌──────────┐
  │ max_heap │  │ min_heap │        │ max_heap │  │ min_heap │
  │ [1,1,3]  │  │   [4]    │   →    │  [1, 1]  │  │  [3, 4]  │
  │  堆顶=3  │  │  堆顶=4  │        │  堆顶=1  │  │  堆顶=3  │
  └──────────┘  └──────────┘        └──────────┘  └──────────┘
  len(max)=3, min=1 ✗               len(max)=2, min=2 ✓

  中位数 = (1 + 3) / 2 = 2.0
```

---

### Step 5: addNum(5)

> 5 > max 堆顶(1) → 放 min_heap
> min 多 1 个 → 把 3 搬到 max

```
  已排序: [1, 1, 3, 4, 5]
                ↑
              中位数=3

  ① 判断: 5 > 1 → 放 min            ② 平衡: min 多 1 个
  ┌──────────┐  ┌──────────┐        ┌──────────┐  ┌──────────┐
  │ max_heap │  │ min_heap │        │ max_heap │  │ min_heap │
  │  [1, 1]  │  │ [3,4,5]  │   →    │[1, 1, 3] │  │  [4, 5]  │
  │  堆顶=1  │  │  堆顶=3  │        │  堆顶=3  │  │  堆顶=4  │
  └──────────┘  └──────────┘        └──────────┘  └──────────┘
  len(max)=2, min=3 ✗               len(max)=3, min=2 ✓

  中位数 = max 堆顶 = 3
```

## ⑥ 一张图总结全流程

```
  addNum(3)    addNum(1)     addNum(4)     addNum(1)     addNum(5)
     │             │              │              │              │
     ▼             ▼              ▼              ▼              ▼
  max:[3]      max:[1]       max:[1,3]     max:[1,1]     max:[1,1,3]
  min:[]       min:[3]       min:[4]       min:[3,4]     min:[4,5]
     │             │              │              │              │
     ▼             ▼              ▼              ▼              ▼
   中=3          中=2           中=3           中=2           中=3
  
  排序:         排序:          排序:          排序:          排序:
  [3]          [1,3]         [1,3,4]       [1,1,3,4]     [1,1,3,4,5]
                ↑             ↑              ↑              ↑
              (1+3)/2         3            (1+3)/2          3
```

## ⑦ 为什么快？

| | 暴力法 | 双堆法 |
|:--|:--|:--|
| 加一个数 | O(1) 扔到数组末尾 | **O(log n)** 堆插入 |
| 查中位数 | O(n log n) 排序 | **O(1)** 读堆顶 |

> 堆的插入和删除都是 O(log n)，就像一个「自动排序的漏斗」，新数进去后自动滚到正确的位置。两个堆顶永远夹着中位数，所以查中位数只要看一眼。

## ⑧ Python 中用的小技巧

Python 的 `heapq` 是**小顶堆**（堆顶最小），但我们需要一个大顶堆。做法很简单——**所有数取负号**：

```
  要放进大顶堆的数: 3, 1, 1
  实际存进 heapq:  -3, -1, -1
  
  heapq 认为堆顶是 -3 (最小)
  我们取负回来:     3   (最大) ← 这就是「较小半的最大值」
```
